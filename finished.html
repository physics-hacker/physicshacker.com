Finished everything?

Did you finish everything? Hungry for more? Here is some thoughts about what to do next:

1. You've just been introduced to computational physics with a programming language that is very similar to C

The simplest thing to do is to continue studying computational physics using the C programming language. There is a short list of introductory-level computational physics books that are written in C. Here is the best, cheapest one:

Computational Physics: Problem Solving with Computers 1st Edition (by Landau & Paez)

Even better there are free video lectures to accompany the text.

2. Now that you are used to a C like programming, perhaps this is a good time to think about C++

C++ offers a way of organizing functions called "classes" that C does not offer. There are also some improvements to the way that matricies are initialized in memory.

If you go to google and search for "learn C++" you will be overwhelmed with the choices, many of them good ones to look into.

Our advice is to check out the website for Physics 6810: Computational Physics at Ohio State University. The first exercise will go into some of the differences between C and C++. Later exercises will go into advanced topics in computational physics.

3. Everyone tells me I should learn python and that C programs are lame

You have two choices: (1) you can learn python and see what all the fuss is about, or (2) you can tell them that C programs run just as fast or faster than python programs (which is completely true).

If you choose the first option, then you can either go to codecombat.com and select python mode, or you can go to the python tutorial for codeacademy.com

When you've finished learning python there are two good computational physics books worth looking at: Computational Physics: Problem Solving with Python by Landau et al. (3rd Edition) which includes free video lectures, and Computational Physics by Mark Newman

Or you can keep on learning computational physics without learning python (see thought #1 above), which is what a lot of computational physicists did before python became popular. You can always learn it later.
