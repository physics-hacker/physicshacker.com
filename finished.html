Finished everything?

Did you finish everything? Hungry for more? Here is some thoughts about what to do next:

1. If you're interested in processing (the language)

If you're interested to learn more about processing (which is the programming language we use here), we highly recommend Daniel Schiffman's book The Nature of Code. The book is freely available in electronic form and the paper version is reasonably priced. We at physicshacker.com have a huge respect for what Daniel Schiffman has done and is doing for the processing community.

2. You've just been introduced to computational physics with a programming language that is very similar to C

The simplest thing to do is to continue studying computational physics using the C programming language. There is a short list of introductory-level computational physics books that are written in C. Here is the best, cheapest one:

Computational Physics: Problem Solving with Computers 1st Edition (by Landau & Paez)

Even better there are free video lectures to accompany the text.

3. Now that you are used to a C like programming, perhaps this is a good time to think about C++ (object-oriented programming)

C++ offers a way of organizing functions called "classes" that C does not offer (sometimes this is also called object-oriented programming).

If you go to google and search for "learn C++" you will be overwhelmed with the choices.

Our advice is to look through the introduction of Daniel Schiffman's book The Nature of Code which describes object-oriented programming and classes and how it is implemented into the processing language. In the exercises we show here we have largely avoided objects and classes for simplicity, but the processing language is perfectly capable of using these features. Even better, the syntax that the processing language uses is very similar, if not the same as used in C++. If you can master the processing language, you will have a much easier time learning C++.

4. Everyone tells me I should learn python and that C programs are lame

You have two choices: (1) you can learn python and see what all the fuss is about, or (2) you can tell them that C programs run just as fast or faster than python programs (which is completely true).

If you choose the first option, then you can either go to codecombat.com and select python mode, or you can go to the python tutorial for codeacademy.com

When you've finished learning python there are two good computational physics books worth looking at: Computational Physics: Problem Solving with Python by Landau et al. (3rd Edition) which includes free video lectures, and Computational Physics by Mark Newman

Or you can keep on learning computational physics without learning python (see thought #1 above), which is what a lot of computational physicists did before python became popular. You can always learn it later.
